HISTORY OF HTML

  • 1989-1991: Tim Berners-Lee invents HTML at CERN to share documents.

  • 1991: The first version of HTML is released with 18 tags.

  • 1993: HTML 2.0 is created to standardize HTML as it grows in use.

  • 1995: HTML 3.2 adds new features like tables and images.

  • 1997: HTML 4.0 introduces scripting, style sheets, and multimedia support.

  • 1999: HTML 4.01 is released with minor updates and bug fixes.

  • 2000s: XHTML is introduced as a stricter version of HTML using XML.

  • 2008: HTML5 development starts, focusing on multimedia and interactivity without plugins

  • 2014: HTML5 becomes the official standard, adding new elements and features.

INTRODUCTION OF HTML

HTML, or HyperText Markup Language, is the standard language used to create and design web pages. It provides the basic structure of a website by using a series of elements or tags that define different parts of the content, such as headings, paragraphs, links, images, and more. Each HTML element is enclosed within angle brackets, with an opening tag to start the element and a closing tag to end it. By combining various HTML elements, developers can create structured and visually appealing web pages that can be displayed in web browsers. HTML is essential for web development, serving as the foundation upon which websites are built.


HTML TAGS


  1. Heading tag

    2. HTML heading tags (h1 to h2) are used to define headings and subheadings within a web page.

    • h1: Represents the main heading of a page. It is typically the most important heading and should be used once per page.

    • h2:Represents a level 2 heading. It is used for subheadings that are important but slightly less significant than h1

    • h3:Represents a level 3 heading. It is used for further subheadings, and the importance decreases with each subsequent level.

    • h4 to h6:These represent decreasing levels of importance for headings. They are used for sub-subheadings and so on, with h6 being the least significant.

  2. Paragraph tag

    • Purpose: The tag is used to define a paragraph of text in an HTML document.

    • Usage: It is one of the most commonly used tags in HTML for structuring and formatting text content.

    • Attributes: The tag does not typically require attributes, but it can accept global attributes like class, id, style, etc., for styling and scripting purposes.

    • Styling: Paragraphs are typically displayed with a space above and below them by default, and their appearance can be further controlled using CSS.

  3. Pre tag

    • Purpose: The pre tag (short for "preformatted") is used to preserve whitespace and line breaks within text, displaying it exactly as it appears in the HTML code.

    • Usage It is primarily used for displaying text where whitespace, line breaks, and formatting are significant and should be maintained.

    • Preservation of Whitespace: Unlike normal HTML text, which collapses multiple spaces and line breaks into a single space or line break, text within pre tags preserves all formatting exactly as it appears in the HTML source code.

    • Attributes: The pre tag supports global attributes like class, id, style, etc., to apply styles or scripts to the content.

  4. Br tag

    • Purpose: The br tag is used to insert a line break in text content within HTML documents.

    • Single Purpose: br is used specifically for line breaks within text, unlike p which defines paragraphs or div which defines divisions.

    • Usage: It is a self-closing tag, meaning it does not have a closing tag (br). It inserts a line break at the current position, without starting a new paragraph.

  5. Hr tag

    • Purpose: The hrtag, short for "horizontal rule," is used to create a thematic break or divider between sections of content within an HTML document.

    • Visual Representation:It renders as a horizontal line across the width of its containing element, typically to visually separate content sections.

    • Attributes:
      width: Specifies the width of the horizontal
      size: Sets the height of the rule as a pixel value.
      color: Defines the color of the rule line.
      noshade: Specifies that the rule should be rendered without a shadow effect (deprecated in HTML5).
